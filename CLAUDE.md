## Project Overview

FRC Team 3996 robot code for the 2026 season. This is a **Python** robotics project using WPILib's command-based framework with CTRE Phoenix 6 swerve drive.

## Commands

```bash
# Install dependencies
uv sync

# Deploy to robot (robot must be connected)
uv run robotpy deploy

# Run simulator
uv run robotpy sim

# Run tests
uv run robotpy test
```

## Architecture

**Entry point**: `robot.py` → `robotcontainer.py` (subsystems + button bindings)

**Design pattern**: WPILib Command-Based (commands2 library). Subsystems extend `commands2.Subsystem`; commands extend `commands2.Command` with `initialize`, `execute`, `end`, `isFinished` methods.

### Subsystems (`subsystems/`)

**Game piece path:** Intake → Conveyor (indexer) → Kicker → Shooter flywheel → Out

| File | Hardware | Physical role |
|------|----------|---------------|
| `command_swerve_drivetrain.py` | CTRE Phoenix 6 swerve, Pigeon 2 gyro (ID 10) | 4-wheel swerve drivetrain. Auto-generated by CTRE Tuner X — be careful editing. |
| `vision.py` | Limelight `limelight-back` | Camera mounted on the robot used for AprilTag-based field localization. MegaTag2 mode; rejects estimates when rotation >720°/s. |
| `shooter.py` | 2× REV SparkMax (IDs 40, 41) | Two motors on a shared flywheel shaft that launches game pieces. Motor 41 follows 40. Closed-loop velocity control. |
| `hood.py` | REV SparkMax (ID 45) | Adjustable plate/angle that sets the launch trajectory of the flywheel. Position-controlled via absolute encoder. |
| `kicker.py` | 2× REV SparkMax (IDs 50, 51) | Two motors on a shared shaft that accelerates the game piece from the conveyor into the shooter flywheel. Motor 51 follows 50. |
| `indexer.py` | REV SparkMax (ID 52) | Single conveyor motor that moves game pieces from the intake up to the kicker. |
| `intake.py` | 2× SparkMax (IDs 55, 56) | ID 55 (brushed) deploys/retracts the intake arm via current control. ID 56 (brushless) spins the intake roller to grab game pieces. |
| `climb.py` | Solenoid (channel 2, CTRE PH) | Pneumatic climber — binary extend/retract. Compressor managed here. |

### Commands (`commands/`)

- `shoot_at_hub.py` — Primary shooting command: distance-based lookup table → RPM + hood angle, auto-aims drivetrain, feeds when within ±200 RPM of target. Hub is at `Translation2d(8.23, 4.115)` meters.
- `climb_commands.py` — `ExtendClimb` / `RetractClimb`
- `shoot_commands.py` — Placeholder, incomplete

### Key Files

- `generated/tuner_constants.py` — CTRE swerve constants (auto-generated by Tuner X, do not hand-edit)
- `modules/limelight.py` — Limelight API wrapper
- `telemetry.py` — SmartDashboard/NetworkTables logging
- `deploy/pathplanner/` — Autonomous path files

## Motor Configuration Pattern

```python
controller = SparkMax(id, SparkMax.MotorType.kBrushless)
config = SparkMaxConfig()
config.smartCurrentLimit(50).idleMode(SparkMaxConfig.IdleMode.kCoast)
config.closedLoop.pid(kP, kI, kD).velocityFF(kFF)
controller.configure(config, SparkMax.ResetMode.kResetSafeParameters, SparkMax.PersistMode.kPersistParameters)
```

## Key Constants

- Max drive speed: 3.79 m/s
- Shooter follower motors spin in the same direction (`follow(id, False)`)
- Max angular velocity: ¾ rotation/s (~2.36 rad/s)
- Joystick deadband: 5% (`0.05`)
- Vision max tag distance: 4.125 m
- Shooter RPM tolerance: ±200 RPM

## Controller Layout

**Driver (Controller 0):** Left stick = translation, Right stick X = rotation, Right trigger = shoot (auto-aim), A = brake, B = point wheels, Left bumper = reset field-centric heading, POV = cardinal directions

**Operator (Controller 1):** POV up/down = extend/retract climber, POV left/right = face left/right
