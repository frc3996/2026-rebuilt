## Project Overview

FRC Team 3996 robot code for the 2026 season. This is a **Python** robotics project using WPILib's command-based framework with CTRE Phoenix 6 swerve drive.

## Commands

```bash
# Install dependencies
uv sync

# Deploy to robot (robot must be connected)
uv run robotpy deploy

# Run simulator
uv run robotpy sim

# Run tests (unit tests only, skip pyfrc)
uv run python -m pytest tests/test_home_hood.py tests/test_auto_tune_hood.py tests/test_home_intake.py tests/test_auto_tune_intake.py -v
```

## Architecture

**Entry point**: `robot.py` → `robotcontainer.py` (subsystems + button bindings)

**Design pattern**: WPILib Command-Based (commands2 library). Subsystems extend `commands2.Subsystem`; commands extend `commands2.Command` with `initialize`, `execute`, `end`, `isFinished` methods.

### Subsystems (`subsystems/`)

**Game piece path:** Intake → Conveyor (indexer) → Kicker → Shooter flywheel → Out

| File | Hardware | Physical role |
|------|----------|---------------|
| `command_swerve_drivetrain.py` | CTRE Phoenix 6 swerve, Pigeon 2 gyro (ID 10) | 4-wheel swerve drivetrain. Auto-generated by CTRE Tuner X — be careful editing. |
| `vision.py` | Limelight `limelight-back` | Camera mounted on the robot used for AprilTag-based field localization. MegaTag2 mode; rejects estimates when rotation >720°/s. Uses NT subscribers (not SmartDashboard). |
| `shooter.py` | 2× REV SparkMax (IDs 47, 43) | Two motors on a shared flywheel shaft that launches game pieces. Motor 43 follows 47. Closed-loop velocity control. |
| `hood.py` | REV SparkMax (ID 45) | Adjustable plate/angle that sets the launch trajectory of the flywheel. Position-controlled via relative encoder. Must be homed before use. Has `set_angle()` with gravity feedforward and dynamic soft limit calibration. |
| `kicker.py` | 2× REV SparkMax (IDs 41, 48) | Two motors on a shared shaft that accelerates the game piece from the conveyor into the shooter flywheel. Motor 48 follows 41. |
| `indexer.py` | REV SparkMax (ID 42) | Single brushed conveyor motor that moves game pieces from the intake up to the kicker. |
| `intake.py` | 2× SparkMax (IDs 46, 44) | ID 46 (brushless) deploys/retracts the intake arm. ID 44 (brushless) spins the intake roller to grab game pieces. Arm must be homed before use. Has `set_angle()` with gravity feedforward and dynamic soft limit calibration. |
| `climb.py` | Solenoid (channel 2, CTRE PH) | Pneumatic climber — binary extend/retract. Disabled — PCM not on CAN bus yet. |

### Commands (`commands/`)

- `home_hood.py` — Homes hood against hard stop using `setVoltage()`, dual stall detection (current + velocity for 5 consecutive cycles), 5s timeout safety.
- `home_intake.py` — Same homing pattern for intake arm.
- `auto_tune_hood.py` — Ziegler-Nichols relay auto-tune for hood position PID.
- `auto_tune_intake.py` — Same Z-N auto-tune for intake arm PID.
- `auto_home.py` — Sequential home hood then intake.
- `shoot_at_hub.py` — Distance-based lookup table → RPM + hood angle, auto-aims drivetrain, feeds when within ±200 RPM of target. Hub is at `Translation2d(8.23, 4.115)` meters.
- `climb_commands.py` — `ExtendClimb` / `RetractClimb` (disabled).

## Motor Configuration Pattern

```python
controller = SparkMax(CANIds.MOTOR_NAME, SparkMax.MotorType.kBrushless)
config = SparkBaseConfig()
config.smartCurrentLimit(50)
config.IdleMode(SparkBaseConfig.IdleMode.kCoast)
config.closedLoop.P(kP, ClosedLoopSlot.kSlot0)
controller.configure(config, rev.ResetMode.kResetSafeParameters, rev.PersistMode.kPersistParameters)
```

## Homing Pattern (hood & intake)

Both hood and intake use relative encoders and must be homed before position control works:
- Drive toward hard stop with `setVoltage()` (not `set()` or current control)
- Stall detection: `getOutputCurrent() > threshold` AND `abs(getVelocity()) < threshold` for N consecutive cycles
- On confirmed stall: `motor.set(0)`, `encoder.setPosition(0)`, mark homed
- Safety timeout prevents infinite stall
- Soft limits disabled before homing, re-enabled after clean completion
- Constants are module-level `UPPER_SNAKE_CASE` with `# TUNE` markers

## Key Constants

- Max drive speed: 3.79 m/s
- Max angular velocity: ¾ rotation/s (~2.36 rad/s)
- Joystick deadband: 5% (`0.05`)
- Vision max tag distance: 4.125 m
- Shooter RPM tolerance: ±200 RPM
- All CAN IDs in `constants.py` (validated on hardware)

## REV API Gotchas (RobotPy)

- `SparkBaseConfig.IdleMode()` uses capital I — not `idleMode()`
- `rev.FeedbackSensor.kPrimaryEncoder` — not `rev.ClosedLoopConfig.FeedbackSensor`
- `rev.ResetMode` / `rev.PersistMode` — not `rev.SparkMax.ResetMode`
- Current control (`kCurrent`) shares PID slot with position — use separate slots or use `setVoltage()`/`motor.set()` for open-loop

## Testing

Tests use `unittest.mock.MagicMock` to mock subsystem objects — no subsystem code changes needed for testing. Commands are instantiated via `__new__` to bypass WPILib C++ `Subsystem.__new__` restriction, then fields are set manually. The `pyfrc_test.py` test has a known fixture issue — run specific test files instead of `tests/` glob.
